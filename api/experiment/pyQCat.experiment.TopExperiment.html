pyQCat.experiment.TopExperiment¶

class pyQCat.experiment.TopExperiment(inst: pyQCat.instrument.instrument_aio.Instrument, *qubits: pyQCat.qubit.qubit.Qubit, correct_dc: Optional[Dict] = None, compensate: Optional[Union[pyQCat.pulse.base_pulse.PulseCorrection, List[pyQCat.pulse.base_pulse.PulseCorrection]]] = None, repeat: int = 1000, bind_dc: bool = True, bind_drive: bool = True, bind_probe: bool = True, discriminator: Optional[Union[pyQCat.tool.iqprobability.IQdiscriminator, List[pyQCat.tool.iqprobability.IQdiscriminator]]] = None, label: Optional[str] = None, multi_readout_channels: Optional[List] = None, schedule_type: str = 'sequence')¶

基类：object

量子比特实验基类

量子比特自动化测试及校准实验顶层接口，包含了各实验的一些通用属性和方法， 建议子类继承时重写 run 方法实现具体的操作

参数

  • inst (Instrument) – 设备对象，封装了实验相关参数的数据库映射对象，集成了各种数据交互操作

  • qubits (Qubit) – 量子比特对象，封装了量子比特的常见参数，自动化校准流程的核心对象

  • correct_dc (Dict) – 字典类型，字典的键为各个比特施加DC电压的通道值，字典的值表示施加到该比特位的DC电压值

  • compensate (Union[PulseCorrection, List[PulseCorrection]]) – 波形校准对象，用来加载畸变数据和由于硬件导致的波形延时数据

  • repeat (int) – 实验重复次数，默认为1000

  • bind_dc (bool) – 是否为一体机绑定当前比特的初始电压

  • bind_drive (bool) – 是否为一体机绑定当前比特的驱动参数（驱动频率、驱动功率）

  • bind_probe (bool) – 是否为一体机绑定当前比特的读取参数（读取频率、读取功率、采样延时、采样带宽）

  • discriminator (Union[IQdiscriminator, List[IQdiscriminator]]) – IQ 分类器，用来将实验测量结果进行分类 从而以不同状态的概率呈现，如 P0-P1 。如果此参数为空，则会用 amp-phase 的方式展示实验测试结果

  • label (str) – 实验标签，可选参数，当需要为实验结果文件夹自定义名称时，可以自定义 label 传入

  • multi_readout_channels (List) – 列表类型，当实验需要多个读取通道时，一般通过自动以读取通道列表传入实验中，默认为空

  • schedule_type (str) – 绘制时序图的类型，决定以什么样的方式绘制波形时序图；默认为 sequence 类型，此外还可以设置为 envelop ，如果为 envelop 类型，则对 XY 线的波形数据使用希尔伯特变换计算解析信号，然后再绘制时序图

  • file (SaveFile) – 实验结果文件操作对象，实现了生成路径，数据保存等一系列功能

  • xy_pulses (Dict) – 字典类型，记录各个比特 XY 线上施加的波形，字典的键为比特对象，字典的值为波形对象或由波形对象构成的列表

  • z_pulses (Dict) – 字典类型，记录各个比特 Z 线上施加的波形，字典的键为比特对象，字典的值为波形对象或由波形对象构成的列表

  • readout_pulses (List) – 列表类型，记录读取通道上施加的波形

play_pulse(name: str, qubit: pyQCat.qubit.qubit.Qubit, pulse: Union[List, pyQCat.pulse.base_pulse.PulseComponent])¶

给XY或Z线添加波形，给 xy_pulses 和 z_pulses 属性更新波形

参数

  • name – 只能设置为 XY 或者 Z ，单比特实验读取通道的波形在实验初始化时已经设置

  • qubit – 比特对象，指明波形施加的对象。

  • pulse – 脉冲对象，封装了加载到硬件上的波形，支持波形列表

引发

  • NameError – 当 name 参数值既不是 XY 也不是 Z

  • TypeError – 当 pulse 参数不是期望的 PulseComponent 时

注意

在调用此方法之前，脉冲对象必须调用它自己的 get_pulse 方法

register(data_type: str = 'amp_phase', enable_one_sweep: bool = False)¶

注册实验

这一步是实验正常运行的关键，它实现了修正波形、注册波形、更新数据库映射等诸多操作。

参数

  • data_type – 实验结果分析类型，可选择 amp-phase 或 I-Q, 默认为 amp-phase 类型。当选择 amp-phase 类型时，实验结果范围幅值和相位信息，且从设置的重复实验次数中取平均值返回； 当选择 I-Q 类型时，会根据 IQ 分类器对 实验结果进行分析，形成当前比特处在0态的概率 P0 和处在1态的概率 P1 的形式范围结果，且会返回与重复实验次数相等 规模的 P0-P1 数据

  • enable_one_sweep – 决定单个波形是否以扫描的形式添加

引发

  • SaveConditionError – 如果注册波形等数据的时候发生了错误

  • ValueError – 如果提供的数据类型既不是 amp-phase 也不是 I-Q 时，将会报错
