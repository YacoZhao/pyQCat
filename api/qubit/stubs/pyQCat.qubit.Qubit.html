import_qubit(json_file: Optional[str] = None)¶

导入比特参数，更新 Qubit 对象属性

当提供了 json_file 路径时，会加载指定 json 文件，将文件中的属性加载到当前的 Qubit 对象上

如果没有提供 json_file ，则会去数据库中查询最近的 QubitStoreDoc 记录， 如果查到结果了，将 QubitStoreDoc 对象参数更新到当前的 Qubit 对象上， 如果没有查到结果，则会创建一个全新的 Qubit 对象

参数

json_file – 指定 josn 文件位置

get_qubit_by_time(create_time, bit: Optional[int] = None)¶

从数据库中加载最新的 QubitStoreDoc 对象，将参数更新到当前的 Qubit 属性中

参数

  • create_time – 创建时间，如 2021-03-26 15:23:38.442

  • bit – 比特编号

export_qubit(export_path: Optional[str] = None)¶

导出 Qubit 参数信息，生成 yaml 文件保存

参数

export_path – 导入比特信息后的 yaml 文件存储路径，默认为 `config/qubit_data’

log_qubit(msg: str, log_path: Optional[str] = None)¶

记录比特日志信息，可以通过此方法记录比特运行状态

参数

  • msg – 记录的日志信息

  • log_path – 日志保存路径

save_qubit()¶

将比特参数信息保存到数据库中，如果比特参数发生了改变，则会在数据库中创建新的 QubitStoreDoc 对象

save_by_yaml(yaml_path: str)¶

解析 yaml 文件中记录 Qubit 参数信息，然后加载到数据库中

参数

yaml_path – yaml 文件路径，例如 {path}/config/q0.yaml

reset()¶

比特初始化操作，如果你意外更新 Qubit 的某些参数，你可以用此方法，让比特恢复成初始状态
