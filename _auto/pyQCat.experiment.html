class ACSpectrum(inst, qubit, init_fringe, delay_list, z_amp_list, T2_bound=500, freq_bound=800, osc_freq_limit=2.5, correct_dc=None, compensate=None, discriminator=None, repeat=1000)¶

基类：object

AC 谱实验

AC谱实验用于计算 flux pulse 的脉冲幅值对比特频率的调制关系。 在判断比特是否可调的实验中，我们通过 DC flux 可以对读取腔产生调制推断出比特频率可调， 所以这里通过 flux pulse ，研究 Z 线上幅值对 qubit 频率的调制情况。实验波形和 Ramsey 实验类似，在两个 π/2pulse 之间间隔一段时间让其演化，在这里不同的是中间演 化期间我们在 Z 线路上施加了一个方波，扫描不同方波的幅值，我们就可以得到 flux pulse 对比特频率的调节关系。实验执行时，可以 不断更新所设fringe的频率，保证ramsey 振荡频率始终处于一小段区间，这样是为了保证振荡频率不会超出采样定理所决定的最高频率 。 用公式拟合 flux pulse 幅度和比特频率的对应关系，即可获取 AC 谱参数。

AC 谱实验运行接口

实验过程中前三个点的 fringe 频率保持不变，因为这个时候实验结果太少，不足以了解调制谱信息； 后面的点根据前三次实验采集的比特频率 fq 和幅值 zamp 映射数据，用二次函数拟合的方式 预测在下一次实验时的比特频率。有了预测的比特频率，我们就可以动态的调整 fringe 频率， 尽可能让拟合频率等于我们设置的 fringe 频率。

from pyQCat.experiment.ac_spectrum import ACSpectrum
from pyQCat.parameters import get_parameters, corrected_dc
from pyQCat.tool.utlis import qarange, connect_server, disconnect_server


def test_ac_spectrum(bit: int, fringe: float, delay_list: list, z_amp_list: list):
    inst = get_parameters("instrument")
    qubit = get_parameters("qubit", bit)
    compensate = get_parameters("compensate", bit)
    dc = corrected_dc(6, sweet_qubit=bit)
    dcm = get_parameters("dcm", bit)
    ac_exp = ACSpectrum(inst, qubit, fringe, delay_list, z_amp_list, correct_dc=dc, compensate=compensate,
                        discriminator=dcm)
    ac_exp.run()

if __name__ == '__main__':
    connect_server(ip="127.0.0.1")
    delay = qarange(200, 500, 5)
    fringe = -30
    zamp_list = qarange(-0.02, 0.4, 0.02)
    test_ac_spectrum(0, fringe, delay, zamp_list)
    disconnect_server()

class AmpOptimize(inst, *qubits, correct_dc=None, compensate=None, repeat=1000, bind_dc=True, bind_drive=True, bind_probe=True, discriminator=None, label=None, multi_readout_channels=None, schedule_type='sequence')¶

幅值优化实验

利用振幅优化实验可以对比特 XYWave 的 Xpi 和 Xpi2 参数进行标定，即优化幅值。 Rabi 实验可以得到初步优化的幅值，而 AmpOptimize 则在此基础上对 Xpi 和 Xpi2 幅值 做更加精细的调节，插入多个待优化波形来放大幅度错误，然后找到 P1 概率最大值所对应的幅值，即为 修正之后的幅值。

  • 根据用户输入参数构建幅值扫描列表 amp_list ；

  • 如果优化 Xpi ，则利用7个重复的 pi_pulse 作为XY线上脉冲；

  • 如果优化的是 Xpi2 ，则利用14个重复的 half_pi_pulse 作为XY线上脉冲；

  • Z线脉冲统一使用Constant；

  • 读取线上脉冲初始化实验时设置，执行实验；

  • 获取实验结果，得到 amp_list 与 amp 或 P1 (qubit处于1态的概率)的对应关系

  • 找出最优幅值，更新 XPi 或 XPi2 即可

AC 串扰实验

假设有两个比特分别为bias qubit和target qubit。 串扰，顾名思义就是bias qubit的flux pulse不仅能调节其自己的频率， 还能影响芯片上其他比特，如target qubit的频率。因此在实验中必须进行串扰实验， 以此计算出相邻比特之间的串扰系数（影响程度）。首先通过AC谱实验测量target qubit 自身flux pulse对比特频率的调制，得到AC谱参数，然后再测量bias qubit的flux pulse 对target qubit的频率调制，拟合得到的MB。有了target qubit自身的调制谱，再结合 bias qubit的flux pulse对target qubit的调制，可以直接计算出Vb相当于多少Vt， 则串扰系数可知。这时要求比特处于对flux pulse幅度比较敏感的位置，因此在测bias qubit 的flux pulse对target qubit的调制时，可以同时在target qubit上施加等长的一定幅度的flux pulse。

run()¶

运行AC串扰实验并获取串扰系数

实验步骤：

  • 根据 freq_offset 属性计算将target_qubit的比特频率下调freq_offset所需要的flux pulse的幅值v_target

  • 用初始的init_v_bias加到偏置比特上，在当前的v_target下进行Ramsye实验，获取比特频率f10

  • 然后明确target_qubit比特频率的调制返回，根据freq_threshold确定

  • 根据串扰公式计算出目标比特进行评率调制所需的z_amp范围

  • 遍历v_bias_list，施加到bias_qubit上，保持v_target施加的target_qubit上，

  • 记录每次实验用AC调制谱参数生成的v_real_list

  • 有了v_bias_list和v_real_list，就可以计算出AC串扰系数了

run()¶

运行DC串扰实验

实验步骤：

  • 获取目标比特的静态工作点电压 target_qubit.dc_max ,记作 v_target

  • 更改编制比特电压，修改 bias_qubit.dc_min 属性

  • 利用实验的 correct_dc 属性，将修改的电压值作用到比特上

  • 目标比特保持当前驱动频率不变，执行Ramsey实验，计算出当前目标比特频率f10

  • 根据DC调制谱参数，计算出将比特调制到当前的f10，所需要施加的DC电压值，即为 v_real

  • 有了 v_real 、 v_target 和 init__v_bias 三个参数之后，可以预测串扰系数为K

  • 根据 freq_threshold 属性确定期望将目标比特频率调制的范围

  • 利用DC谱参数的和预计的DC串扰系数确定编制比特扫描的电压列表，记作 v_bias_list

  • 遍历 v_bias_list ,利用预测的DC串扰系数K值，计算当前 v_bias 可能对应的 v_predict

  • 在用DC调制谱公式计算当前 v_predict 下的比特频率，调整比特频率，保证比特一直处于静态工作点

  • 执行Ramsey实验，计算比特频率f10

  • 根据DC谱公式，计算比特频率真实的电压 v_real_list

  • 根据 v_bais_list 和 v_real_list 即可求出DC串扰系数

Reference arXiv:1907.04818v1 (L.DiCarlo) Time-domain characterization and correction of on-chip distortion of control pulses in a quantum processor

flux pulse 信号从 AWG 产生到到达比特，中途经历了很多线缆，微波器件和连接头，在这些位置都有可能产生衰减、反射造成 波形畸变失真，我们可以将比特视为一个示波器，在 flux pulse 线上施加阶跃脉冲(加幅值)引起比特频率变化，由于 flux pulse 和比特频率存在对应关系，如果我们能够测出在阶跃脉冲施加过程中比特频率随时间的变化，我们就能计算出阶跃脉冲经过线路达到比特 的响应情况，然后通过反卷积的方法进行校准。

执行畸变实验

固定两个 half pi pulse 之间的间隔 separation 不变，改变 Z 线路方波的宽度 width (从0扫描到 separation/2)， 对于每一个 width 值，分别使第二个 half pi pulse 为 X/2 和 Y/2，然后执行 singleshot 测量，获取一组比特处于0 态和1态下的概率记为 [P0x, P1x] ， [P0y, P1y] 获取畸变参数 sigam_x 和 sigam_y ， 计算公式为:

\[ \begin{align}\begin{aligned}sigma_x = 1 - 2 * P1x\\sigma_y = 1 - 2 * P1y\end{aligned}\end{align} \]

  • 对 sigma_x 做傅里叶变换，得到频谱分量最大值对应的频率，记为 IF ，该频率即为 Z 波 形方波最后平稳饱和时对应的振荡频率（比特在该处频率与工作点的差值 detune）

  • 根据相位微分得到频率，该频率的物理意义是 Z 波形方波在上升沿至平稳饱和过程中每一时刻处比特频率与工作点频率的差值

class TopExperiment(inst, *qubits, correct_dc=None, compensate=None, repeat=1000, bind_dc=True, bind_drive=True, bind_probe=True, discriminator=None, label=None, multi_readout_channels=None, schedule_type='sequence')¶

基类：object

量子比特实验基类

量子比特测试及校准实验顶层接口，包含了量子比特测试实验所必须的属性和方法，如果你想实现新的实验，建议你继承此类， 同时重写 run() 方法实现具体的操作

参数

  • inst (Instrument) – 设备对象，封装了实验相关参数的数据库映射对象，集成了各种数据交互操作

  • qubits (Qubit) – 量子比特对象，封装了量子比特的待表征参数（如读取频率、读取功率、驱动频率、驱动功率、门操作脉冲参数等），它是 量子比特测试及校准的核心对象

  • correct_dc (Dict) – 字典类型，字典的键为各个比特施加DC电压的通道值，字典的值表示施加到该比特位的DC电压值

  • compensate (Union[PulseCorrection, List[PulseCorrection]]) – 波形校准对象，用来加载畸变数据和由于硬件导致的波形延时数据, 执行单比特实验时，我们只需要输入待测试比特对应的校准对象即可，而执行多比特实验时，我们需要输入每个比特对象的波形校准对象构成 列表传入

  • repeat (int) – 实验重复次数，默认为1000。该属性可以理解为小循环，量子比特校准实验在每一个具体的扫描参数下都会重复执行 repeat 次，比特具备一定数量的小循环次数，才可以统计出当前比特在 0 态和 1 态下的概率 P0 和 P1

  • bind_dc (bool) – 是否为一体机绑定当前比特的初始电压，默认为 True

  • bind_drive (bool) – 是否为一体机绑定当前比特的驱动参数（驱动频率、驱动功率），默认为 True

  • bind_probe (bool) – 是否为一体机绑定当前比特的读取参数（读取频率、读取功率、采样延时、采样带宽），默认为 True

  • discriminator (Union[IQdiscriminator, List[IQdiscriminator]]) – IQ 分类器，用来将实验测量结果进行分类 从而以不同状态的概率呈现，如 P0-P1 。如果此参数为空，则会用 amp-phase 的方式展示实验测试结果

  • label (str) – 实验标签，可选参数，当需要为实验结果文件夹自定义名称时，可以自定义 label 传入

  • multi_readout_channels (List) – 列表类型，当实验需要多个读取通道时，一般通过自定义读取通道列表传入实验中，默认为空

  • schedule_type (str) – 绘制时序图的类型，决定以什么样的方式绘制波形时序图；默认为 sequence 类型，此外还可以设置为 envelop ，如果为 envelop 类型，则对 XY 线的波形数据使用希尔伯特变换计算解析信号，然后再绘制时序图

  • id – 实验数据库映射类 ExperimentDoc 的数据库存储 id

  • analysis (TopAnalysis) – 用于绑定实验结果分析类

  • qubit (Qubit) – 当实验输出比特对象仅有一个（即单比特实验），该属性会被赋值，否则该属性为空

  • file (SaveFile) – 实验结果文件操作对象，实现了生成路径，数据保存等一系列功能

  • xy_pulses (Dict) – 字典类型，记录各个比特 XY 线上施加的波形，字典的键为比特对象，字典的值为波形对象或由波形对象构成的列表

  • z_pulses (Dict) – 字典类型，记录各个比特 Z 线上施加的波形，字典的键为比特对象，字典的值为波形对象或由波形对象构成的列表

  • readout_pulses (List) – 列表类型，记录读取通道上施加的波形

register(data_type='amp_phase', enable_one_sweep=False)¶

注册实验

这一步是实验正常运行的关键，在子实验执行流程中，必须调用此接口，它实现的功能主要有：

  • 生成 Z 线路补偿脉冲

  • 注册 XY 线和 Z 线波形至一体机

  • ValueError – 如果提供的数据类型既不是 amp-phase 也不是 I-Q 时，将会报错

幅值优化实验

利用振幅优化实验可以对比特 XYWave 的 Xpi 和 Xpi2 参数进行标定，即优化幅值。 Rabi 实验可以得到初步优化的幅值，而 AmpOptimize 则在此基础上对 Xpi 和 Xpi2 幅值 做更加精细的调节，插入多个待优化波形来放大幅度错误，然后找到 P1 概率最大值所对应的幅值，即为 修正之后的幅值。

  • 根据用户输入参数构建幅值扫描列表 amp_list ；

  • 如果优化 Xpi ，则利用7个重复的 pi_pulse 作为XY线上脉冲；

  • 如果优化的是 Xpi2 ，则利用14个重复的 half_pi_pulse 作为XY线上脉冲；

  • Z线脉冲统一使用Constant；

  • 读取线上脉冲初始化实验时设置，执行实验；

  • 获取实验结果，得到 amp_list 与 amp 或 P1 (qubit处于1态的概率)的对应关系

  • 找出最优幅值，更新 XPi 或 XPi2 即可

AC 谱实验

AC谱实验用于计算 flux pulse 的脉冲幅值对比特频率的调制关系。 在判断比特是否可调的实验中，我们通过 DC flux 可以对读取腔产生调制推断出比特频率可调， 所以这里通过 flux pulse ，研究 Z 线上幅值对 qubit 频率的调制情况。实验波形和 Ramsey 实验类似，在两个 π/2pulse 之间间隔一段时间让其演化，在这里不同的是中间演 化期间我们在 Z 线路上施加了一个方波，扫描不同方波的幅值，我们就可以得到 flux pulse 对比特频率的调节关系。实验执行时，可以 不断更新所设fringe的频率，保证ramsey 振荡频率始终处于一小段区间，这样是为了保证振荡频率不会超出采样定理所决定的最高频率 。 用公式拟合 flux pulse 幅度和比特频率的对应关系，即可获取 AC 谱参数。

  • 根据 freq_offset 属性计算将target_qubit的比特频率下调freq_offset所需要的flux pulse的幅值v_target

  • 用初始的init_v_bias加到偏置比特上，在当前的v_target下进行Ramsye实验，获取比特频率f10

  • 再根据target_qubit的AC谱参数计算不考虑串扰所需要的flux pulse幅值v_real

  • 根据v_target和v_real可以预估出一个串扰系数k0

  • 然后明确target_qubit比特频率的调制返回，根据freq_threshold确定

  • 根据串扰公式计算出目标比特进行评率调制所需的z_amp范围

  • 根据前面计算出来的k值，推测bias_qubit上施加的z_amp范围，记为v_bias_list

  • 遍历v_bias_list，施加到bias_qubit上，保持v_target施加的target_qubit上，

  • 记录每次实验用AC调制谱参数生成的v_real_list

  • 有了v_bias_list和v_real_list，就可以计算出AC串扰系数了

run()¶

运行DC串扰实验

实验步骤：

  • 获取目标比特的静态工作点电压 target_qubit.dc_max ,记作 v_target

  • 更改编制比特电压，修改 bias_qubit.dc_min 属性

  • 利用实验的 correct_dc 属性，将修改的电压值作用到比特上

  • 目标比特保持当前驱动频率不变，执行Ramsey实验，计算出当前目标比特频率f10

  • 根据DC调制谱参数，计算出将比特调制到当前的f10，所需要施加的DC电压值，即为 v_real

  • 有了 v_real 、 v_target 和 init__v_bias 三个参数之后，可以预测串扰系数为K

  • 根据 freq_threshold 属性确定期望将目标比特频率调制的范围

  • 利用DC谱参数的和预计的DC串扰系数确定编制比特扫描的电压列表，记作 v_bias_list

  • 遍历 v_bias_list ,利用预测的DC串扰系数K值，计算当前 v_bias 可能对应的 v_predict

  • 在用DC调制谱公式计算当前 v_predict 下的比特频率，调整比特频率，保证比特一直处于静态工作点

  • 执行Ramsey实验，计算比特频率f10

  • 根据DC谱公式，计算比特频率真实的电压 v_real_list

  • 根据 v_bais_list 和 v_real_list 即可求出DC串扰系数
