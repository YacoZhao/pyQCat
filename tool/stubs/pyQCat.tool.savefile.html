class SaveFile(root, sample='', qubit='', task='', savemode='.dat')¶

基类：object

文件存储功能类

参数

  • root (str) – 根路径，pyQCat 的实验模板中默认中从 Qubit 对象的 sample 属性获取， 必须设置

  • sample (str) – 比特标签，默认为空字符串

  • qubit (str) – 比特字符串，一般以 q 加上具体的比特位构成，如 q0 、 q1 ，默认为空字符串

  • task (str) – 任务描述，一般用于任务描述

  • savemode (str) – 数据存储格式, 默认为 .dat ， 你还可以设置为 .txt 、 .ini 等

初始化 SaveFile 对象时，会默认将输出的路径参数组合成路径地址保存至 path 属性中，当你输入路径为

  • root: D:\Qubit\D4\2021-08-18

  • sample: author

  • qubit: q0

  • task: XEB

则拼接后的路径为 D:\Qubit\D4\2021-08-18\author\q0\XEB , 它将赋值给 path 属性

sample = qubit.sample + '/' + datetime.now().strftime("%Y-%m-%d")
qs = 'q' + str(qubit.bit)

file_sys = SaveFile(
    root=sample,
    qubit=qs,
    savemode='.dat'
)

property dirs¶

set(directory, time=False)¶

在初始 path 的基础上新增子文件夹

参数

  • directory (str) – 新增的文件名

  • time (bool) – 是否添加以时间戳命名的子文件目录，默认为 False , 即不添加时间

假设现在有 SaveFile 的 path 属性为 D:\Qubit\D4\2021-08-18\author\q0\XEB ，如果我们希望新增子目录， 可以通过当前的 set 方法，使用方法如下：

file_sys = SaveFile(
    root=sample,
    qubit=qs,
    savemode='.dat'
)

file_sys.set('test', time=False)
file_sys.dirs
# 输出为：D:\Qubit\D4\2021-08-18\author\q0\XEB\test

file_sys.set('test', time=True)
file_sys.dirs
# 输出为： D:\Qubit\D4\2021-08-18\author\q0\XEB\test\2021-08-28 12.45.33
返回类型

None

get()¶

已废弃，使用 dirs 代替

reset()¶

重置文件存储类

当你需要更新你的存储路径时，不需要重新定义 SaveFile 对象，你只需要使用这个方法，即可将与此对象绑定 存储路径 dirs 属性清空，但仍然会持有 path 属性，你可再次调用 set() 方法，新增你期望的路径

save(*data, name=None)¶

保存数据

参数

  • *data – 待存储的数据

  • name (Optional[str]) – 文件名称

name 属性为你期望保存的文件名称，我们在初始化方法中提供了 savemode 属性供你设置存储文件类型，我们默认为 .dat 文件，如果你没有提供了 name 值，那么它将以数字 0/1/2/3/4... 来给文件命名

file_sys.dirs
# D:\Qubit\D4\2021-08-18\author\q0\XEB\test\2021-08-28 12.45.33

file_sys.save(data, "measure1")
# data数据存储的绝对路径为 D:\Qubit\D4\2021-08-18\author\q0\XEB\test\2021-08-28 12.45.33\measure1.dat

file_sys.save(data)
# data数据存储的绝对路径为 D:\Qubit\D4\2021-08-18\author\q0\XEB\test\2021-08-28 12.45.33\0.dat

file_sys.save(data)
# data数据存储的绝对路径为 D:\Qubit\D4\2021-08-18\author\q0\XEB\test\2021-08-28 12.45.33\1.dat
