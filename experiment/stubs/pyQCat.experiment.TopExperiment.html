pyQCat.experiment.TopExperiment¶

class TopExperiment(inst, *qubits, correct_dc=None, compensate=None, repeat=1000, bind_dc=True, bind_drive=True, bind_probe=True, discriminator=None, label=None, multi_readout_channels=None, schedule_type='sequence')¶

基类：object

量子比特实验基类

量子比特测试及校准实验顶层接口，包含了量子比特测试实验所必须的属性和方法，如果你想实现新的实验，建议你继承此类， 同时重写 run() 方法实现具体的操作

参数

  • inst (Instrument) – 设备对象，封装了实验相关参数的数据库映射对象，集成了各种数据交互操作

  • qubits (Qubit) – 量子比特对象，封装了量子比特的待表征参数（如读取频率、读取功率、驱动频率、驱动功率、门操作脉冲参数等），它是 量子比特测试及校准的核心对象

  • correct_dc (Dict) – 字典类型，字典的键为各个比特施加DC电压的通道值，字典的值表示施加到该比特位的DC电压值

  • compensate (Union[PulseCorrection, List[PulseCorrection]]) – 波形校准对象，用来加载畸变数据和由于硬件导致的波形延时数据, 执行单比特实验时，我们只需要输入待测试比特对应的校准对象即可，而执行多比特实验时，我们需要输入每个比特对象的波形校准对象构成 列表传入

  • repeat (int) – 实验重复次数，默认为1000。该属性可以理解为小循环，量子比特校准实验在每一个具体的扫描参数下都会重复执行 repeat 次，比特具备一定数量的小循环次数，才可以统计出当前比特在 0 态和 1 态下的概率 P0 和 P1

  • bind_dc (bool) – 是否为一体机绑定当前比特的初始电压，默认为 True

  • bind_drive (bool) – 是否为一体机绑定当前比特的驱动参数（驱动频率、驱动功率），默认为 True

  • bind_probe (bool) – 是否为一体机绑定当前比特的读取参数（读取频率、读取功率、采样延时、采样带宽），默认为 True

  • discriminator (Union[IQdiscriminator, List[IQdiscriminator]]) – IQ 分类器，用来将实验测量结果进行分类 从而以不同状态的概率呈现，如 P0-P1 。如果此参数为空，则会用 amp-phase 的方式展示实验测试结果

  • label (str) – 实验标签，可选参数，当需要为实验结果文件夹自定义名称时，可以自定义 label 传入

  • multi_readout_channels (List) – 列表类型，当实验需要多个读取通道时，一般通过自定义读取通道列表传入实验中，默认为空

  • schedule_type (str) – 绘制时序图的类型，决定以什么样的方式绘制波形时序图；默认为 sequence 类型，此外还可以设置为 envelop ，如果为 envelop 类型，则对 XY 线的波形数据使用希尔伯特变换计算解析信号，然后再绘制时序图

  • id – 实验数据库映射类 ExperimentDoc 的数据库存储 id

  • analysis (TopAnalysis) – 用于绑定实验结果分析类

  • qubit (Qubit) – 当实验输出比特对象仅有一个（即单比特实验），该属性会被赋值，否则该属性为空

  • file (SaveFile) – 实验结果文件操作对象，实现了生成路径，数据保存等一系列功能

  • xy_pulses (Dict) – 字典类型，记录各个比特 XY 线上施加的波形，字典的键为比特对象，字典的值为波形对象或由波形对象构成的列表

  • z_pulses (Dict) – 字典类型，记录各个比特 Z 线上施加的波形，字典的键为比特对象，字典的值为波形对象或由波形对象构成的列表

  • readout_pulses (List) – 列表类型，记录读取通道上施加的波形

config_file_sys(name=None, display=True)¶

设置实验分析结果文件的保存路径

参数

  • name (Optional[str]) – 自定义实验名称，默认为空。如果不为空，则结果文件会在最后新开辟一个以 name 命名的新文件夹来存放实验结果

  • display (bool) – 是否将实验结果保存路径以日志的形式输出

路径名称为: sample/time/qubit/lable/time-name

  • sample: 从比特对象(Qubit)的 sample 属性中记录

  • time: 生成当前时间戳，转化为 年月日 字符串

  • qubit: 从当前 qubits 属性中生成的关于实验目标比特的字符串，如果 q0 、 q0q1

  • label: 如果初始化实验的时候没有提供 label 属性，则默认将当前类名幅值给它

  • time-name: time 为当前时间戳转化为 年月日时分秒 的字符串，如果当前方法提供了 name 属性，则会 在 time 后面用 - 进行连接,生成 time-name 的形式

案例

假设我们有：
sample: D:\Qubit\D4
time: 2021-08-18
qubit: q0
lable: XEB
name: None
则最终生成的存储文件路径为: ``D:\Qubit\D41-08-18\q0\XEB``

你可以非常方便的在实验逻辑执行之前调用此方法，如：

self.config_file_sys()
self.config_file_sys(name = "test", display = False)  # 生成文件目录，但不打印日志
返回类型

None

save_data(*args, file_name=None)¶

保存自定义的数据到当前实验预先生成的结果路径下

参数

  • *args – 期望保存的数据

  • file_name (Optional[str]) – 期望保存后的文件名称，默认为空，如果为空，则用 0、1、2 数字序列生成文件名

案例

self.save_data(data1)  # 输出 0.dat
self.save_data(data2)  # 输出 1.dat
self.save_data(data3, file_name = "test")  # 输出 test.dat
引发

SaveFileError – 保存数据的过程发成异常，并提示错误原因，通常情况下异常原因是在调用此方法之前没有初始化 file 属性

返回类型

None

save_parameters(**kwargs)¶

保存实验参数到本地目录

调用此方法会将与当前实验绑定的实验参数 MeasureAIODoc 和量子比特对象 QubitDoc 以文件的形式存储 到提前生成的实验结果路径中。其中测量参数 MeasureAIODoc 会保存成 json 文件， Qubit 参数 会保存成 txt 文件

参数

**kwargs – 实验测量参数，传入字典类型的数据。

注意

传入的 kwargs 参数必须为 MeasureAIODoc 对象的属性

返回类型

None

play_pulse(name, qubit, pulse)¶

给 XY 或 Z 线绑定波形

该方法主要完成线路波形的绑定，并未真正将波形参数下传至数据库，当前操作会给 self.xy_pulses 和 self.z_pulses 属 性绑定好输出波形，改方法只支持 XY 线 和 Z 线，读取通道施加波形此方法并不支持

参数

  • name (str) – 只能设置为 XY 或者 Z ，单比特实验读取通道的波形在实验初始化时已经设置

  • qubit (Qubit) – 比特对象，指明脉冲施加的对象。

  • pulse (Union[List, PulseComponent]) – 脉冲对象，封装了加载到硬件上的波形，支持波形列表

引发

  • NameError – 当 name 参数值既不是 XY 也不是 Z

  • TypeError – 当 pulse 参数不是期望的 PulseComponent 对象时

注意

在调用此方法之前，脉冲对象必须提前调用自己的 get_pulse 方法

案例

self.play_pulse("XY", self.qubit, xy_pulse_list)
self.play_pulse("Z", self.qubit, z_pulse_list)
返回类型

None

acquire_pulse(acq_pulse, qubit=None)¶

设置读取波形

执行单比特实验室，通常情况你不需要调用此方法，因为我们在初始化实验的时候就通过 init_readout 方法绑定了读取通道波形。 当多比特实验，施加联合读取波形或者施加新的读取波形时，你可以调用此方法。

参数

  • acq_pulse (PulseComponent) – 读取波形对象

  • qubit (Optional[Qubit]) – 施加的 Qubit 对象，可以用来获取读取通道值，默认为空，如果为空，则从 实验的 qubits[0] 属性中加载读取通道

案例

amp_list, IF_list, index_list = [], [], []
for qubit in self.qubits:
    amp_list.append(qubit.union_readout.amp)
    IF_list.append(qubit.union_readout.probe_IF)
    index_list.append(qubit.union_readout.index)
union_readout_channel = self.qubits[0].readout_channel
self.set_multiple_IF(*IF_list, channel=union_readout_channel)
self.set_multiple_index(*index_list, channel=union_readout_channel)
union_readout_pulse = AcquireSine(acq_time, amp_list, IF_list)
union_readout_pulse.get_pulse()
self.readout_pulses.append(union_readout_pulse)
# 施加联合读取波形
self.acquire_pulse(union_readout_pulse)
返回类型

None

set_readout_delay(*delay, channel=None)¶

设置读取通道波形的脉冲触发延时

读取通道上的脉冲必须等到 XY 线和 Z 线波形施加完成才可以激发，因此需要给读取通道施加触发延时。

你一般不需要调用此方法，因为我们在 self.register() 方法中几乎为所有的实验设置了默认的触发延时。

参数

  • *delay – 延时时间

  • channel (Optional[int]) – 通道值，默认为空，从实验的 qubit 属性和 qubits 属性中加载

案例

self.set_readout_delay(150, channel = 1)
返回类型

None

register(data_type='amp_phase', enable_one_sweep=False)¶

注册实验

这一步是实验正常运行的关键，在子实验执行流程中，必须调用此接口，它实现的功能主要有：

  • 生成 Z 线路补偿脉冲

  • 注册 XY 线和 Z 线波形至一体机

  • 自动设置读取通道脉冲的触发延时

  • 扫描参数整理

参数

  • data_type (str) – 实验结果分析类型，可选择 amp-phase 或 I-Q, 默认为 amp-phase 类型。当选择 amp-phase 类型时，实验结果范围幅值和相位信息，且从设置的重复实验次数中取平均值返回； 当选择 I-Q 类型时，会根据 IQ 分类器对 实验结果进行分析，形成当前比特处在0态的概率 P0 和处在1态的概率 P1 的形式范围结果，且会返回与重复实验次数相等 规模的 P0-P1 数据

  • enable_one_sweep (bool) – 决定单个波形是否以扫描的形式添加

引发

  • SaveConditionError – 如果注册波形等数据的时候发生了错误

  • ValueError – 如果提供的数据类型既不是 amp-phase 也不是 I-Q 时，将会报错

案例

if self.discriminator is not None:
    data_type = "I_Q"
else:
    data_type = "amp_phase"
self.register(data_type)
返回类型

None

plot_schedule(xy_index=0, is_save=True)¶

绘制波形时序图

你可以通过 xy_index 来选择你希望看到的波形时序图，前提条件必须保真 xy_index 小于扫描波形列表的长度。

参数

  • xy_index (int) – 波形索引，默认为0，即选取属性 self.xy_pluses 中第一个元素， XY 线和 Z 线公用一个索引，保证一致性

  • is_save (bool) – 是否保存图片，默认保存

注意

此方法调用必须在 self.play_pulse() 方法之后，否则会接受不到脉冲波形数据

案例

self.play_pulse("XY", self.qubit, detune_pulse_list)

if self.discriminator is not None:
    data_type = "I_Q"
else:
    data_type = "amp_phase"

self.register(data_type)

self.analysis = APEAnalysis(self.id, data_type, detune_list, parameters, self.discriminator, self.file)

# plot schedule
self.plot_schedule(xy_index=0)
返回类型

None

init_readout(time=None, amp=None, IF=None, channel=None)¶

初始化读取模块，为读取线添加波形

一般情况下，你不需要调用此方法，我们在执行单比特实验流程中，默认调用了此方法。当你执行单比特实验流程时， 需要修改读取脉冲波形，你可以调用此方法实现修改操作。

参数

  • time (Optional[float]) – 波形的宽度，默认为 None，从 qubit 属性中获取

  • amp (Optional[float]) – 波形的幅值，默认为 None，从 qubit 属性中获取

  • IF (Optional[float]) – 波形中频，默认为 None，从 qubit 属性中获取

  • channel (Optional[int]) – 读取通道，默认为 None，从 qubit 属性中获取

案例

time = 1500.0
amp = 0.5
IF = 600.0
channel = 1
self.init_readout(time = time, amp = amp, IF = IF, channel = channel)
返回类型

None
