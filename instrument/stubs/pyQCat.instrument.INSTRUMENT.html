pyQCat.instrument.Instrument¶

class Instrument(support_qubit=8)¶

基类：object

设备类

量子测控一体机通常有四个功能模块，分别为：

  • 集成 AWG 模块: 用于施加 Z 线路波形。在代码中，我们用 Z_flux_control 控制 AWG 模块的输出波形

  • 集成电压源 DC 模块: 用于给比特施加 DC 电压，调制比特的工作点。在代码中，我们用 Z_dc_control 控制 DC 模块的输出波形

  • 集成矢量源 DA(RF) 模块: 用于施加 XY 线路波形。在代码中，我们用 XY_control 控制 RF 模块的输出波形

  • 集成微波测量收发 ADDA 模块: 用于施加读取线路波形，并接受反馈波形。在代码中，我们用 Read_out_control 控制 ADDA 模块的输出波形

我们通常会从配置文件路径下的 json 文件或者数据库中加载设备对象，该初始设备对象中定义了一体机各个模块的模板参数。

Instrument 中记录了所有模块的模板参数，所以我们需要从初始模板参数中提取实验所需要的参数，我们可以根据模块名和模块下对应的通道获取 对应的实验模板参数，当前类下的 select() 可以帮你实现这个功能。

对一体机的各个模块，我们进行了通道的限制，当你为模块绑定参数时，如果通道值不在规定的范围内，将会报错。

一体机型号为 8bit 时：

  • Z_dc_control 通道范围为 1-8

  • Z_flux_control 通道范围为 1-8

  • XY_control 通道范围为 5-20

  • Readout_control 通道范围为 1-4

一体机型号为 30bit 时：

  • Z_dc_control 通道范围为 1-80

  • Z_flux_control 通道范围为 1-80

  • XY_control 通道范围为 9-40

  • Readout_control 通道范围为 1-8

通过 select() 方法，会生成当前实验参数模板，封装到数据库映射类 MeasureAIODoc 中，这时候我们只需要修改实验模板中的属性 即可， 而 Instrument 类就是帮你完成实验参数的设置及获取功能。

参数

  • support_qubit (int) – 一体机型号，目前支持8位和30位，默认为8位

  • _kernel (Kernel) – 测量参数封装类，设备对象与数据库映射之间的关联类

  • _sweep (List) – 实验最终施加到设备上的扫描参数

  • instrument_aio (InstrumentAIODoc) – 设备数据库映射类

  • is_connect (bool) – 设备是否连接的标志位

select(**kwargs)¶

初始化测量参数模板

我们通常会从配置文件路径下的 json 文件或者数据库中加载设备对象，该初始设备对象中定义了一体机各个模块的模板参数。我们需要从 初始模板中提取我们需要的属性组合成当前实验的输入参数。

而 select 方法就是帮助你完成这样的需求，你只需要输入指定模块对应的通道值，它就可以为你生成你想要实验参数模板文件。

参数

**kwargs – 字典类型，键为指定模块名，值为通道值， 模块名可以为 Z_dc_control 、 Z_flux_control 、 XY_control 、 Readout_control

引发

InstrumentError – 配置文件属性为空，设置参数之前必须加载配置文件

案例

z_dc_channels = [0, 1, 2, 3, 4, 5]
z_flux_channels = [3]
xy_channels = [5]
readout_channels = [1]
# 抽取指定通道下的模板参数
self.inst.select(
    Z_dc_control=z_dc_channels,
    Z_flux_control=z_flux_channels,
    XY_control=xy_channels,
    Readout_control=readout_channels
)
组合成当前实验的模板参数为

说明

上面的实验参数模板就是当前实验的输入参数模板，我们会将它记录到数据库的 MeasureAIODoc 对象中，你只需要修改 这个实验参数模板中的参数，即可完成实验参数下传。

None

set_sync_period(value)¶

设置非反馈模式下的执行周期

在进行重复实验时，每个周期内执行一次施加波形，执行实验的操作，非反馈模式下可以用这个属性 保证一体机各个模块周期相同。

参数

value –

一体机施加波形周期

  • 最小值为10

  • 最大值为1000

  • 步长为5，设置的周期值必须可能被 5ns 整除

  • 单位: 微秒(us)

案例

self.set_sync_period(100)
返回类型

None

set_period(module, channel, value)¶

仪器模块下为各个模块单独设置脉冲周期

为一体机的 AWG/RF/ADDA 模块设置脉冲周期，区别于 set_sync_period 方法，此方法是在仪器模式下， 单独设置某个模块的执行周期，而 set_sync_period 是非反馈模式下设备小循环实验的执行周期。一般实验 我们不需要考虑仪器模式下的模块周期值。

参数

  • module (str) – 模块名，可以设置为 Z_flux_control 、 XY_control 、 Read_out_control 、 Trig_out_control

  • channel (int) – 通道值

  • value (int) – 周期值，范围为 10-1000，单位：微秒（us）

案例

self.set_period("XY_control", 5, 100)
返回类型

None

set_trig_mode(module, channel, mode)¶

仪器模块下为各个模块单独设置触发模式

为一体机 AWG/RF/ADDA 模块设置触发模式。区别于 set_sync_trig_mode 方法，此方法是在仪器模式下， 单独设置某个模块的触发模式，而 set_sync_trig_mode 是非反馈模式下设备各个模块的触发模式。一般实验 我们不需要考虑仪器模式下的各个模块单独的触发模式。

参数

  • module (str) – 模块名，可以设置为 Z_flux_control 、 XY_control 、 Read_out_control 、 Trig_out_control

  • channel (int) – 通道值

  • mode (int) – 触发模式 ( 0/1/2/3 )

案例

self.set_trig_mode("XY_control", 5, 1)
返回类型

None

sweep_delay(module, channel, start=None, stop=None, step=None, points=None, repeat=1000, index=0)¶

一体机 AWG/RF/ADDA 模块设置扫描延时

当施加的脉冲波形需要扫描触发延时时，需调用此方法，如果 RB 实验中每个深度下的读取波形触发延时都是不一样的， 所以必须为读取通道波形施加扫描延时。

参数

  • module (str) – 模块名，可以设置为 Z_flux_control 、 XY_control 、 Read_out_control

  • channel (int) – 通道

  • start – 触发延时扫描参数的开始时间，范围为 0-100000ns

  • stop – 触发延时扫描参数的结束时间，范围为 0-100000ns

  • step – 触发延时扫描参数的步长，范围为 0-100000ns

  • points – 可迭代的对象，如果没有给出起始、结束、步长三个参数，可以直接传入延时列表交给 points 参数

  • repeat – 重复次数，默认为1000，只有在非反馈模式的时候有效

  • index – 延时列表的索引

引发

ArgumentsError – 如果传入的延时参数有误

返回类型

List

返回

扫描参数

注意

步长必须为 5ns 的整数倍

案例

self.play_pulse("XY", self.qubit, pulse_list)
self.inst.sweep_delay("Readout_control", self.qubit.readout_channel,
                      points=self._pulse_time_list, repeat=self.repeat)
